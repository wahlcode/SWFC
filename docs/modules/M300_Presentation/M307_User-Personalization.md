\# M307 – User Personalization / Experience



\---



\## 🎯 Zweck



M307 steuert die \*\*individuelle Darstellung und Nutzung von SWFC pro Benutzer\*\*.



Es ist zuständig für:



\- persönliche Dashboard-Zusammenstellung

\- Auswahl sichtbarer Inhalte

\- individuelle Startseite

\- UI-Personalisierung

\- Benutzerpräferenzen in der Darstellung



\---



\## 🧠 Kernsatz



> M307 bestimmt, \*\*was ein Benutzer sieht und wie er das System persönlich nutzt\*\*



\---



\## Inhalt



\- Dashboard-Konfiguration

\- Widget-Auswahl und -Reihenfolge

\- modulbezogene Sichtbarkeit

\- persönliche UI-Präferenzen

\- individuelle Startseiten

\- Anzeigeeinstellungen



\---



\## Struktur



M307 besteht logisch aus:



\- M307.01 – Dashboard Configuration  

\- M307.02 – Widget Management  

\- M307.03 – Visibility \& Preferences  

\- M307.04 – User UI Settings  



\---



\## Abgrenzung



\### M307 ist zuständig für



\- persönliche Darstellung

\- Dashboard-Inhalte

\- UI-Personalisierung

\- Benutzerpräferenzen



\---



\### M307 ist NICHT zuständig für



\- Theme / Farben → M106

\- Sprache → M306

\- Berechtigungen → M806

\- Navigation-Grundstruktur → M301

\- fachliche Daten → M200

\- Analytics-Inhalte → M211



\---



\## Verknüpfungen



\### M102 – Users

\- speichert Benutzer

\- kann Präferenzen referenzieren



\---



\### M301 – AppShell

\- liefert Struktur

\- M307 bestimmt Inhalte innerhalb dieser Struktur



\---



\### M106 – Theme

\- bestimmt Darstellung

\- M307 bestimmt Inhalte



\---



\### M306 – Localization

\- bestimmt Sprache

\- M307 bestimmt Sichtbarkeit



\---



\### M211 – Analytics

\- liefert Dashboard-Daten

\- M307 bestimmt, welche davon angezeigt werden



\---



\## Regeln



\### 1. Keine Fachlogik

M307 entscheidet nur über Darstellung, nicht über Inhalte.



\---



\### 2. Keine Sicherheitslogik

Sichtbarkeit ersetzt keine Berechtigungen.



\---



\### 3. Benutzerzentriert

Alle Einstellungen sind benutzerbezogen.



\---



\### 4. Persistenz

Benutzerpräferenzen müssen gespeichert werden.



\---



\## Zielzustand



M307 ist korrekt umgesetzt, wenn:



\- Benutzer ihr Dashboard selbst konfigurieren können

\- Widgets individuell auswählbar sind

\- Startseiten personalisiert sind

\- Darstellung pro Benutzer unterschiedlich sein kann

\- System trotz Personalisierung konsistent bleibt



\---



\## Bedeutung



M307 ist entscheidend für:



\- Benutzerfreundlichkeit

\- Effizienz im Alltag

\- Akzeptanz des Systems

\- individuelle Arbeitsweisen
